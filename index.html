The 5-hour clock only starts when you do

Claude Max gives you a generous usage window that resets every five hours. The catch: the timer starts at your first message, not at dawn.

Sit down at 10:00, hit your limit at noon — and now you're staring at the wall until 15:00. Five full hours, counted from when you showed up. That's the bug. This is the fix.

One morning, on air

A GitHub Actions cron checks in every 15 minutes — in the cloud, so your machine sleeps as late as you do. The schedule below is what actually happens.

08:00

The robot grabs the mic

The previous window has expired, so the automation sends Claude a one-line "buongiorno" on the cheapest model. The 5-hour clock starts ticking — while you're still in bed.

10:00

You sit down to work

The window has already been running for two hours. You spend the morning burning tokens like a champ.

12:00

You hit the limit

It happens to the best of us. But the clock started at 08:00, not 10:00 — so the reset lands at 13:00, not 15:00.

13:00

Back on air

Fresh window, fresh tokens. You waited one hour instead of three. The cycle repeats itself, dynamically, all day long.

Two hours of waiting, gone. Every single day.

Loud on the surface, boring underneath

It's ~150 lines of dependency-free Python on a GitHub Actions cron. The boring parts are the point:

  • Never double-sends

    Before saying anything, it checks Anthropic's OAuth usage endpoint. If a 5-hour window is already active — because you're working, or it already fired — it stays quiet. No wasted tokens, no duplicate windows.

  • Dynamic, not a fixed alarm

    No hardcoded 8 AM. The cron probes every 15 minutes and fires the moment the previous window expires, so the cycle always restarts from your last session — around the clock.

  • Costs almost nothing

    The wake-up call is a single one-turn message on haiku, the cheapest model. It nibbles a crumb of quota to unlock the whole loaf earlier.

  • Fails safe

    If the usage endpoint is down or changes shape, it falls back to a committed state.json timestamp and refuses to send more than once per window. Worst case: it does nothing.

On air in five minutes

You need a Claude Pro or Max subscription and a GitHub account. That's it.

Fork the repo

Your fork carries the workflow. GitHub runs it for you, free, forever.

gh repo fork marcozambrella/gooodmorning-vietnam --clone

Mint a token

On the machine where you're already logged into Claude Code. The token lives for a year.

claude setup-token   # copy the printed token

Set the secret & go

Drop the token into your fork's Actions secrets. The cron takes over from there.

gh secret set CLAUDE_CODE_OAUTH_TOKEN --repo <you>/gooodmorning-vietnam
